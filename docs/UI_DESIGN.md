# SilverConnect Global — UI 设计规格

> 本文档供 Claude Design 生成 UI 视觉稿使用。所有屏幕均面向 **60+ 老年用户**，对比度按 **WCAG 2.1 AAA** 执行（高于 AA），其余项符合 AA。

---

## 0. 设计原则

1. **老年友好优先** — 大字号、高对比、宽间距、单列布局、明确反馈。
2. **温暖而克制** — 用线条插画讲故事，不堆装饰；插画为信息服务，不是壁纸。
3. **看得见的信任** — 价格透明（含税）、Provider 资质徽章、托管支付提示常驻。
4. **多国多语** — 同一组件支持 AU / CN / CA，EN / ZH 文案等长。
5. **AI 永远在场（C 端）** — 客户端与 Provider 端右下角悬浮聊天按钮可呼出；管理员后台不挂载（管理员走内部工单）。

## 1. 设计系统（Design Tokens）

### 1.1 色板

#### 浅色（默认）

| Token | 值 | 用途 |
|---|---|---|
| `--brand-primary` | `#1F6FEB` (深蓝) | 主按钮、链接 |
| `--brand-primary-hover` | `#1858C4` | 主按钮悬浮 |
| `--brand-accent` | `#F59E0B` (温暖橙) | 强调、徽章、收藏 |
| `--success` | `#16A34A` | 已支付、已完成 |
| `--warning` | `#F59E0B` | 待处理 |
| `--danger` | `#DC2626` | 紧急、争议、取消 |
| `--text-primary` | `#0F172A` | 正文（对比度 ≥ 16:1） |
| `--text-secondary` | `#475569` | 次要文本 |
| `--bg-base` | `#FFFFFF` | 页面背景 |
| `--bg-surface` | `#F8FAFC` | 卡片底 |
| `--border` | `#E2E8F0` | 分隔线 |

#### 深色（可选，跟随系统 — **非默认**）

> ⚠️ **浅色为默认且首选模式**。60+ 老年用户晶状体黄化，深底白字会产生光晕散射（halation），看起来发糊。深色仅在用户系统偏好为深色时跟随，不主动推送，不作为出图主稿。

| Token | 值 |
|---|---|
| `--brand-primary` | `#5B9CFF` |
| `--text-primary` | `#F1F5F9` |
| `--text-secondary` | `#94A3B8` |
| `--bg-base` | `#1E293B`（不用近黑 `#0F172A`，过深加重眩光） |
| `--bg-surface` | `#334155` |
| `--border` | `#475569` |

> 不使用纯灰文字于白底；正文对比度 **≥ 7:1（WCAG AAA）**，大字 ≥ 4.5:1。深色模式按钮加 `--success`/`--danger` 增亮 15% 保持对比。蓝色卡片在深色下用 `#1E3A8A` 等低饱和深蓝，避免高饱和亮蓝刺眼。

#### 出图默认

所有屏幕优先出**浅色版**作为主交付稿。深色版本仅在需要展示跟随系统能力时单独出一两屏示意，不全套出。

#### 前景-背景配色契约（Foreground-on-Background Contract）⚠️ 强制

每个文本/图标节点的颜色由其**所在容器的背景明度**唯一决定，**不按整屏判断、按节点判断**。嵌套容器各自独立。

| 容器背景明度 | 主文字 | 次文字 | 占位符 | 图标 | 边框 | 链接 |
|---|---|---|---|---|---|---|
| 深 (L* < 50) | `#F1F5F9` | `#CBD5E1` | `#94A3B8` | `#F1F5F9` | `#475569` | `#93C5FD` |
| 浅 (L* ≥ 50) | `#0F172A` | `#475569` | `#94A3B8` | `#0F172A` | `#E2E8F0` | `#1F6FEB` |

按钮：填充色 + 反相文字。主蓝填充 → 白字；红色危险 → 白字；Ghost 按钮按所在底色判断。

**禁止组合**：
- ❌ `#0F172A` 字在 `#1E293B`/`#334155`/`#1F6FEB` 等任何深底上
- ❌ `#F1F5F9` 字在 `#FFFFFF`/`#F8FAFC`/`#EFF6FF` 等任何浅底上
- ❌ 状态徽章在深底用浅绿/浅黄底叠深字（深底徽章应为半透明白底 `rgba(255,255,255,0.12)` + 白字）

**对比度硬指标（WCAG AAA）**：正文 ≥ 7:1，大字 (≥22px) ≥ 4.5:1，图标与按钮 ≥ 4.5:1。每屏出图前用拾色器抽样自检。

### 1.2 字体

- 字族：`Inter`（拉丁）+ `Noto Sans SC`（中文），系统兜底。
- 基准字号：**18px**（不是 14/16）。
- 标尺：H1 32 / H2 26 / H3 22 / Body 18 / Small 16。
- 行高：1.6。
- 字重：Body 400，强调 600，标题 700。

### 1.3 尺寸 & 触控

- 触控目标最小 **48×48px**（按钮、复选框、链接热区）。
- 主按钮高 **56px**，圆角 12px。
- 表单输入框高 **56px**，圆角 12px，1.5px 边框。
- 卡片内边距 24px，圆角 16px，`box-shadow: 0 2px 8px rgba(15,23,42,.06)`。
- 页面最大内容宽度 720px（移动优先 + 桌面单列居中）。
- 间距等差：4 / 8 / 12 / 16 / 24 / 32 / 48。
- 焦点环：2px 实线 `--brand-primary`，外侧 4px 同色 20% 透明度光晕；深底用 `#93C5FD`。键盘 Tab 必现。

### 1.4 图标

- 线性，stroke-width 2px，尺寸 24/32/48。
- 关键操作配图标 + 文字（不只图标）。

### 1.5 状态反馈

- 成功 / 错误 / 加载 — 全部使用图标 + 颜色 + 文本三重信号（不只颜色）。
- 加载状态使用骨架屏，不用 spinner。
- Toast 停留 5 秒（老人需要更长阅读时间）。

### 1.6 响应式断点

| 断点 | 宽度 | 布局 |
|---|---|---|
| `sm` | < 640px | 单列，底部 Tab 栏，Header 折叠为 Logo + 国家/语言双按钮 + 头像（不收成汉堡，老人找不到） |
| `md` | 640–1024px | 单列居中，顶部 Header，无底部 Tab |
| `lg` | ≥ 1024px | 单列居中（最大 720px 内容），左/右留白 |

> 不做多列瀑布流。老人不需要 dashboard 式密集布局。
> 例外：管理员后台 (`/admin`) 桌面优先，可用多列表格。

### 1.7 动效

- 按下反馈：scale(0.97) + 80ms。
- 页面切换：fade 200ms，无滑动方向（避免眩晕）。
- 不使用 parallax、视差、自动轮播。
- 尊重 `prefers-reduced-motion`，全部降为透明度切换。

### 1.8 插画系统（Line-Illustration System）⭐

> 让产品有体温的关键。所有插画统一**手绘感线条 + 单色暖色填充**风格。

### 1.8.1 风格规则

- **线条**：stroke-width 2.5px，圆头圆角（`stroke-linecap: round`），略带不规则弧度（不是几何完美），传递手绘温度。
- **填充**：仅使用 1–2 个色块，从以下 4 色挑选：
  - `#FDE68A` 暖黄（毛衣、阳光）
  - `#FECACA` 暖粉（脸颊、围裙）
  - `#BFDBFE` 浅蓝（衣物、窗户）
  - `#BBF7D0` 浅绿（植物、围巾）
- **不使用**：渐变、阴影、写实质感、3D。
- **比例**：人物头身比 1:3（略 Q 版），更亲切；老人略驼背、Provider 站姿挺拔，做出区分。
- **道具**：每个角色配 1–2 件叙事道具（拐杖/茶杯/拖把/工具箱/小盆栽）。

### 1.8.4 微动画（Micro-interaction）

让插画"活"起来的关键 — 不要静态图：

- **首页 hero S1**：茶杯热气循环上升（SVG path 2.5s loop）。
- **AI 头像 C9**：眨眼每 4s 一次，说话时嘴部小幅波动。
- **支付成功 S5**：✓ 描边 0.6s 一次性绘制。
- **空状态 S3**：小猫尾巴 3s 一次轻摆。
- **加载中**（替代骨架屏的 hero 区）：小伴机器人头顶三个点跳动。
- **完成评价**：5 星点亮时星星弹跳 + 王阿姨竖大拇指 0.5s。

> 全部尊重 `prefers-reduced-motion` — 关闭后插画静止。

### 1.8.5 插画 token 与位置规则

| 位置 | 插画类型 | 尺寸 |
|---|---|---|
| 页面 hero 区 | 场景插画 | 高 200px，居右或居中 |
| 空状态 | 场景插画 | 高 160px，居中 |
| 服务大类卡 | 单角色半身 | 96×96px，卡片左侧 |
| Provider 默认头像 | 单角色头像 | 圆形 80px |
| AI 气泡左侧 | C9 头像 | 48px |
| 按钮内 | ❌ 不放插画 | — |

**克制规则**：每屏插画总面积 ≤ 25%，不与表单/CTA 抢焦点。表单页面（支付确认、地址输入）**不放场景插画**，只保留小图标。

## 2. 全局组件

### 2.1 顶部导航 Header

布局示意（同一行五块，左到右）：

```
[Logo: SilverConnect]   [🇦🇺 AU ▾]   [中文 ▾]   [🔍]   [👤 / 登录]
```

- 高度 80px，吸顶。
- **品牌名 "SilverConnect" 在任何 locale 下保持原样，永不翻译**（曾被误译为"银联" / UnionPay，严重事故，禁止再发生）。
- **国家切换器**：宽 88px、高 56px。一个国旗 emoji + 一个**两位国家码** + 下拉箭头。**禁止"AU AU UZ"这类多码堆叠**。下拉项展示国旗 + 全名（Australia / 中国 / Canada）。
- **语言切换器**：宽 88px、高 56px。当前 locale 自称（`English` / `中文`），下拉项使用各自语言自称（不翻译）。
- 国家与语言相互独立：切国家不改语言，切语言不改国家。
- 未登录时 `[👤]` 位置显示为「Sign in / 登录」按钮（主色填充）。
- 移动端（< 640px）：Logo + 国家 + 语言 + 头像四块同行，搜索图标隐藏，搜索框降为页内组件（不收成汉堡菜单 — 老人找不到）。

### 2.2 底部导航（移动端）

5 个 Tab，每个 64px 高，图标 32px：
`首页` · `服务` · `预订` · `消息` · `我的`

### 2.3 AI 悬浮按钮

- 右下角，距边 24px，圆形 64×64px，主色背景，AI 图标 + 「问一下」三字（中文模式）/「Ask AI」（英文模式）。
- 紧急关键词触发后，按钮变 `--danger` 红色，弹出全屏紧急联系卡（见 §7.4）。

### 2.4 通用按钮

| 类型 | 样式 |
|---|---|
| Primary | 主色填充，白字，56px 高，圆角 12px |
| Secondary | 白底，2px 主色边框，主色字 |
| Ghost | 无边框，主色字，仅次要操作 |
| Danger | `--danger` 填充，用于取消/删除（带二次确认） |

## 5. 页面：预订流程

### 5.1 BookingForm（4 步进度条）

顶部步骤指示：`①服务 → ②时间 → ③地址 → ④确认`，当前步高亮。

#### Step 1 — 选服务包
- 单选大卡（不下拉）：「2 小时基础清洁」「4 小时深度清洁」「全屋整理」
- 每卡显示：时长、范围、价格（A$45 含 GST）。

#### Step 2 — 选时间

### 5.2 BookingStatusFlow

水平 5 节点：`待支付 → 已确认 → 进行中 → 待确认完成 → 已完成`

| 进行中 | In progress | status=IN_PROGRESS |

## 10. 多语言适配规则

- **EN 与 ZH 是两个独立 locale，永不在同屏并排显示。** 用户在 Header 切换，任意时刻屏幕只渲染一种语言。出图时每屏必须分 EN / ZH 两个独立变体提交，禁止 `Hi, Helen 你好，海伦` 或 `Map 地图` 这种拼接形式。
- **品牌名 `SilverConnect` 永不翻译**（曾被误译为"银联" / UnionPay，严重事故）。
- 所有按钮文字按中文 4 字、英文 12 字符预留宽度，不能裁剪。
- **数字、货币符号、单位、地名拉丁拼写、人名拉丁拼写在中文 locale 下保持原字符**，不译为"澳元/公里"等，也不在后面加中文括号注释。
- 货币符号语义须正确：`A$` = 澳元 AUD（**不是美元 USD**），`¥` = 人民币 CNY，`C$` = 加元 CAD。中文 locale 保留符号，必要时只补"（人民币）"等一次性说明，不全文替换为中文币种名。
- 金额格式两 locale 一致：`A$1,240.00`，千分位用半角逗号。
- 国别紧急号码硬编码三套（见 §7.4）。
