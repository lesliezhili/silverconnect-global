# 产品需求（PRD）

## 1. 愿景

为澳大利亚、中国、加拿大的老年人提供单一可信平台，发现、预订并安全支付经审核的上门服务——具备老年友好 UI、多语言、24/7 AI 协助。

## 2. 用户画像

| 角色 | 目标 | 痛点 |
|---|---|---|
| **老年客户**（60+） | 预订靠谱帮手、UI 简单、付款安全 | 技术焦虑、语言、诈骗 |
| **家属看护** | 替父母下单、监督上门 | 协调、距离 |
| **服务提供者** | 持续接单、及时拿钱 | 空档、争议、回款慢 |
| **运营/管理员** | 解争议、审核 Provider、监控安全 | 欺诈、合规、规模 |

## 3. 范围

### MVP 范围内（已交付）

- 3 国：AU、CN、CA。币种：AUD、CNY、CAD。
- 25+ 项服务，5 大类：清洁、烹饪、园艺、个人护理、维修。
- 客户注册、登录、资料、地址、支付方式、收藏。
- Provider 注册、资质上传、可用时段、屏蔽时间、服务区域、徽章。
- 预订引擎：单次 + 循环（周/双周/月）、提醒、变更。
- Stripe 收款 + 托管、退款、Connect payout。
- 评价、评分、商家回应、评价举报。
- 争议（含证据 + 留言）、事件报告、安全标记。
- Provider 合规材料。
- AI 客服：会话、意图、知识库、模板、紧急检测。
- EN + ZH 本地化。
- 管理员后台。

### MVP 范围外（后续）

- 原生 App（React Native）。
- 视频问诊。
- 保险对接。
- AU/CN/CA 之外的国家。
- 法语。

## 4. 用户故事（Epic）

### E1 — 发现 & 预订
- 作为客户，我能浏览本国服务，看到本币含税价。
- 作为客户，我能按服务、地点、评分、语言筛选 Provider。
- 作为客户，我能预订单次或循环服务（具体日期时间）。
- 作为客户，我能在政策范围内改约/取消。

### E2 — 支付
- 作为客户，我可以刷卡支付；款项进入托管直至服务完成。
- 作为客户，我可以按取消政策申请退款。
- 作为 Provider，服务完成后通过 Stripe Connect 收款。

### E3 — 信任与安全
- 作为用户，我能附证据提交争议。
- 作为用户，我能提交安全事件；按严重度通知管理员。
- 作为 Provider，我能上传合规材料（无犯罪证明、急救等）。

### E4 — 反馈
- 作为客户，我能给完成的预订打分写评价。
- 作为 Provider，我能回应评价。

### E5 — AI 协助
- 作为用户，我能 24/7 用 EN/ZH 与 AI 助手聊天。
- AI 检测到紧急情况立刻显示紧急联系。
- AI 能创建/修改/取消预订，回答价格/服务问题。

### E6 — 管理员
- 作为管理员，我能审核争议、安全标记、事件报告。
- 作为管理员，我能验证 Provider、批准合规材料。

## 5. 功能性需求（节选）

状态图例：✅ 代码已强制 · 🟡 部分/弱强制 · ⚠️ 仅文档约定，API 层未强制。

| ID | 需求 | 状态 | 说明 |
|---|---|---|---|
| FR-01 | 价格必须含本地税（AU 10% GST、CN 0% VAT、CA 13% HST）。 | ✅ | `service_prices.price_with_tax` + `lib/pricing.ts`。 |
| FR-02 | 距开始 > 24h 取消：全退；≤ 24h 按政策。 | 🟡 | 仅 AI agent (`ai_customer_service.py`) 中强制；`DELETE /api/bookings/[id]` 直接置状态，无政策检查。 |
| FR-03 | 预订变更需提前 ≥ 2h 提出。 | ⚠️ | 代码中无强制；仅 KB/帮助文案约定。 |
| FR-04 | 客户确认完成后释放托管（或 T+48h 自动释放）。 | 🟡 | DB 函数 `release_escrow(booking_id)` 已存在；无定时器在 T+48h 触发。仅手动释放。 |
| FR-05 | AI 必须检测紧急关键字并显示对应国家紧急联系。 | ✅ | `components/AIChat.tsx` 的 `EMERGENCY_KEYWORDS` + `checkForEmergency`。 |
| FR-06 | RLS 必须阻止用户读他人预订、支付、消息。 | ✅ | RLS 策略见 `lib/schema.sql` 与各迁移。 |
| FR-07 | Provider 收款需已验证的 Stripe Connect 账号。 | 🟡 | `service_providers.stripe_connect_id` 列已在；payout 端点未硬阻无/未验证 id。 |

## 6. 非功能性需求

| 维度 | 目标 |
|---|---|
| 无障碍 | WCAG 2.1 AA；触控目标 ≥ 48px；高对比；屏幕阅读器标签。 |
| 性能 | Lighthouse Perf ≥ 0.9 于 `/`、`/services`、`/bookings`（取自 `lighthouserc.json`）。 |
| 可用性 | 月度 99.5%。 |
| 安全 | 符合 OWASP Top 10；secrets 仅在 env；所有 PII 表启 RLS。 |
| 隐私 | 与 GDPR 对齐的数据导出/删除（CN PIPL 后续）。 |
| 本地化 | 客户可见处全部 EN + ZH。 |

## 7. 成功指标

- 每周预订量（按国别）。
- 90 日复购率。
- 平均评分 ≥ 4.5。
- 完成订单争议率 < 2%。
- AI 化解率（无人工解决比例）。
- 支付成功率 ≥ 98%。
