# SilverConnect Global — UI 设计规格

> 本文档供 Claude Design 生成 UI 视觉稿使用。所有屏幕均面向 **60+ 老年用户**，需符合 WCAG 2.1 AA。

---

## 0. 设计原则

1. **老年友好优先** — 大字号、高对比、宽间距、单列布局、明确反馈。
2. **少即是多** — 每屏只解决一个核心任务，避免装饰性元素。
3. **看得见的信任** — 价格透明（含税）、Provider 资质徽章、托管支付提示常驻。
4. **多国多语** — 同一组件支持 AU / CN / CA，EN / ZH 文案等长。
5. **AI 永远在场** — 右下角悬浮聊天按钮，所有页面可呼出。

## 1. 设计系统（Design Tokens）

### 1.1 色板

#### 浅色（默认）

| Token | 值 | 用途 |
|---|---|---|
| `--brand-primary` | `#1F6FEB` (深蓝) | 主按钮、链接 |
| `--brand-primary-hover` | `#1858C4` | 主按钮悬浮 |
| `--brand-accent` | `#F59E0B` (温暖橙) | 强调、徽章、收藏 |
| `--success` | `#16A34A` | 已支付、已完成 |
| `--warning` | `#F59E0B` | 待处理 |
| `--danger` | `#DC2626` | 紧急、争议、取消 |
| `--text-primary` | `#0F172A` | 正文（对比度 ≥ 16:1） |
| `--text-secondary` | `#475569` | 次要文本 |
| `--bg-base` | `#FFFFFF` | 页面背景 |
| `--bg-surface` | `#F8FAFC` | 卡片底 |
| `--border` | `#E2E8F0` | 分隔线 |

#### 深色（可选，跟随系统）

| Token | 值 |
|---|---|
| `--brand-primary` | `#5B9CFF` |
| `--text-primary` | `#F1F5F9` |
| `--text-secondary` | `#94A3B8` |
| `--bg-base` | `#0F172A` |
| `--bg-surface` | `#1E293B` |
| `--border` | `#334155` |

> 不使用纯灰文字于白底；最低对比度 7:1。深色模式按钮加 `--success`/`--danger` 增亮 15% 保持对比。

### 1.6 响应式断点

| 断点 | 宽度 | 布局 |
|---|---|---|
| `sm` | < 640px | 单列，底部 Tab 栏，Header 折叠汉堡 |
| `md` | 640–1024px | 单列居中，顶部 Header，无底部 Tab |
| `lg` | ≥ 1024px | 单列居中（最大 720px 内容），左/右留白 |

> 不做多列瀑布流。老人不需要 dashboard 式密集布局。

### 1.7 动效

- 按下反馈：scale(0.97) + 80ms。
- 页面切换：fade 200ms，无滑动方向（避免眩晕）。
- 不使用 parallax、视差、自动轮播。
- 尊重 `prefers-reduced-motion`，全部降为透明度切换。

### 1.2 字体

- 字族：`Inter`（拉丁）+ `Noto Sans SC`（中文），系统兜底。
- 基准字号：**18px**（不是 14/16）。
- 标尺：H1 32 / H2 26 / H3 22 / Body 18 / Small 16。
- 行高：1.6。
- 字重：Body 400，强调 600，标题 700。

### 1.3 尺寸 & 触控

- 触控目标最小 **48×48px**（按钮、复选框、链接热区）。
- 主按钮高 **56px**，圆角 12px。
- 表单输入框高 **56px**，圆角 12px，1.5px 边框。
- 卡片内边距 24px，圆角 16px，`box-shadow: 0 2px 8px rgba(15,23,42,.06)`。
- 页面最大内容宽度 720px（移动优先 + 桌面单列居中）。
- 间距等差：4 / 8 / 12 / 16 / 24 / 32 / 48。

### 1.4 图标

- 线性，stroke-width 2px，尺寸 24/32/48。
- 关键操作配图标 + 文字（不只图标）。

### 1.5 状态反馈

- 成功 / 错误 / 加载 — 全部使用图标 + 颜色 + 文本三重信号（不只颜色）。
- 加载状态使用骨架屏，不用 spinner。
- Toast 停留 5 秒（老人需要更长阅读时间）。

## 2. 全局组件

### 2.1 顶部导航 Header

```
[Logo SilverConnect]  [国家 🇦🇺 AU ▾]  [语言 中文 ▾]  [搜索 🔍]  [我 👤]
```

- 高度 80px，吸顶。
- 国家/语言切换器为大按钮（≥ 56px 宽），下拉里展示国旗 + 全名。
- 未登录时 `[我]` 显示为「登录 / 注册」按钮（主色填充）。

### 2.2 底部导航（移动端）

5 个 Tab，每个 64px 高，图标 32px：
`首页` · `服务` · `预订` · `消息` · `我的`

### 2.3 AI 悬浮按钮

- 右下角，距边 24px，圆形 64×64px，主色背景，AI 图标 + 「问一下」三字（中文模式）/「Ask AI」（英文模式）。
- 紧急关键词触发后，按钮变 `--danger` 红色，弹出全屏紧急联系卡（见 §7.4）。

### 2.4 通用按钮

| 类型 | 样式 |
|---|---|
| Primary | 主色填充，白字，56px 高，圆角 12px |
| Secondary | 白底，2px 主色边框，主色字 |
| Ghost | 无边框，主色字，仅次要操作 |
| Danger | `--danger` 填充，用于取消/删除（带二次确认） |

## 5. 页面：预订流程

### 5.1 BookingForm（4 步进度条）

顶部步骤指示：`①服务 → ②时间 → ③地址 → ④确认`，当前步高亮。

#### Step 1 — 选服务包
- 单选大卡（不下拉）：「2 小时基础清洁」「4 小时深度清洁」「全屋整理」
- 每卡显示：时长、范围、价格（A$45 含 GST）。

#### Step 2 — 选时间
- 月历视图（不下拉），可用日绿色，不可用灰色。
- 选定日后下方出现时段大按钮：`9:00` `11:00` `14:00`（每个 80×60px）。
- 切换：`单次 / 每周 / 每两周 / 每月` — 4 个 tab。

#### Step 3 — 地址
- 已存地址列表（每条 80px 高，单选）。
- 「+ 新增地址」按钮在底部。
- 显示距离 Provider 的预估车程。

#### Step 4 — 确认
- 价格明细表（服务费 / 平台费 / GST / 总计），字号大。
- 取消政策提示（高亮框）：「距开始 > 24 小时取消可全额退款」。
- 「使用 Apple Pay / Google Pay / 信用卡」分三块支付方式。
- 底部 sticky：`确认并支付 A$60.50` 主按钮。

### 5.2 BookingStatusFlow

水平 5 节点：`待支付 → 已确认 → 进行中 → 待确认完成 → 已完成`

- 当前节点放大 1.4 倍，主色填充；已过节点 `--success`；未到灰色。
- 每节点下方显示时间戳。

## 6. 页面：我的预订 `/bookings`

- Tab：`即将进行` / `历史` / `循环`
- 卡片：日期大字（24 号 周三 14:00）、Provider 头像 + 名、服务类型、状态徽章、金额。
- 卡片右下「改约」「取消」「联系 Provider」三按钮（≥ 48px 高）。

---

## 6.5 页面：预订详情 `/bookings/[id]`

- 顶部状态徽章（大字 24px）+ BookingStatusFlow（§5.2 横向时间线）。
- 服务卡：服务名 / 日期时间 / 时长 / Provider 头像 + 名（可点跳详情）。
- 地址卡：地址全文 + 「打开地图」按钮。
- 价格明细可折叠表（默认展开）。
- 操作区（按状态显示不同按钮）：
  - `待支付` → `去支付` 主按钮 + `取消` 次按钮
  - `已确认` → `改约` `取消` `联系 Provider`
  - `进行中` → `联系 Provider` `报告安全问题`（红字）
  - `待确认完成` → `确认完成并评价` 主按钮
  - `已完成` → `再次预订` `查看评价`
- 底部「我有问题」弱按钮 → 进入争议流程（§7.9）。

## 6.6 通知中心 `/notifications`

- Tab：`全部` / `预订` / `支付` / `系统`
- 每条 80px 高：图标 + 标题 + 一行摘要 + 相对时间 + 未读小红点。
- 点击跳转到对应详情页。
- 顶部「全部标为已读」次按钮。
- 通知设置入口在右上角齿轮图标，跳 §7.6 的通知设置子页（短信/邮件/Push 三组开关）。

### 7.4 紧急模式（FR-05）
检测到 `救命/120/急救/胸痛/摔倒` 等关键词后：
- 全屏红底卡片：「请立即拨打 **000**（澳洲）/ **120**（中国）/ **911**（加拿大）」
- 大号一键拨打按钮（120px 高）。
- 「同时通知我的紧急联系人」次按钮。

## 7.5 页面：登录 / 注册 AuthModal

- 全屏模态（移动端），桌面端 480px 居中卡。
- 标题：「欢迎回来」/「创建账号」，副标题一句解释「安全预订上门服务」。
- 输入框：邮箱 + 密码，56px 高，下方一行小字「至少 8 位」。
- 主按钮：`登录` / `注册` 全宽 56px。
- 分隔线下方：`使用 Google 继续` / `使用 Apple 继续`（次按钮）。
- 底部「忘记密码？」「我没有账号 → 注册」大号链接。
- **不使用图形验证码**（老人难辨识）；用邮箱验证码 + rate limit。

## 7.6 页面：个人资料 `/profile`

- 顶部头像 96px + 姓名 + 国家 + 语言切换大按钮。
- 列表项（每条 72px 高，带图标 + 右箭头）：
  - 我的地址
  - 支付方式
  - 紧急联系人
  - 收藏的 Provider
  - 通知设置
  - 帮助与客服
  - 退出登录（红色字）

## 7.7 页面：支付方式 `/profile/payment`

- 已绑定卡列表：每张展示卡 brand 图标 + 后 4 位 + 到期 + 默认徽章 + 删除。
- 「+ 添加新卡」按钮触发 Stripe Elements 嵌入表单（不跳出本站）。
- 顶部信息条：🔒「您的卡号由 Stripe 加密存储，本站从不接触」。

## 7.8 评价 FeedbackForm

触发：预订状态变为「待确认完成」后，主页顶部出现条幅「请评价 周三的清洁服务 →」。

- 大号 5 星点击区（每星 64×64px）。
- 多选 chip：`准时` `专业` `干净` `态度好` `价格合理`（选中变主色）。
- 可选文本框（最少 0 字，不强制）。
- 「上传照片」按钮（最多 5 张）。
- 提交后跳「服务完成 → 释放托管支付」确认页（FR-04）。

## 7.9 争议提交 DisputeForm（E3）

入口：预订详情页底部「我有问题」次按钮。

- 单选大卡选择争议类型：`没按时来` / `服务不完整` / `损坏物品` / `其他`。
- 文字描述（必填，≥ 20 字）。
- 上传证据（照片/视频/录音，最多 10 个）。
- 期望结果单选：`重做` / `部分退款` / `全额退款`。
- 提示框：「提交后管理员将在 48 小时内回复，托管款项暂不释放」。

## 7.10 安全事件 SafetyReport

入口：AI 聊天紧急模式 + 预订详情页「报告安全问题」红色链接。

- 严重度三选大按钮：🟢 一般 / 🟡 中等 / 🔴 紧急。
- 选「紧急」立即跳转 §7.4 紧急联系卡。
- 其他级别填表：发生时间 + 描述 + 证据 + 是否需要警方介入复选框。

## 7.11 通用错误 / 空状态

| 场景 | 视觉 |
|---|---|
| 网络断开 | 居中插画 64px + 「连不上网」+ 「重试」大按钮 |
| 接口错误 | 同上 + 「我们这边出了点问题，请稍后重试」+ 错误码小字 |
| 列表为空 | 浅色插画 + 一句解释 + 一个 CTA（如「浏览服务 →」） |
| 支付失败 | 红色顶部条 + 「卡被拒，原因：xxx」+ 「换张卡」「联系银行」按钮 |
| 加载中 | 骨架屏（卡片轮廓灰块）— 不用 spinner |

## 9. 页面：管理员后台 `/admin`

- 左侧固定窄侧栏（80px，仅图标）：争议 / 安全标记 / Provider 审核 / 合规材料 / 数据。

## 10. 多语言适配规则

- 所有按钮文字按中文 4 字、英文 12 字符预留宽度，不能裁剪。
- 中文模式数字保持西文字符（`A$60.50`，不转「六十澳元五角」）。
- 国别紧急号码硬编码三套（见 §7.4）。

---

## 11. 无障碍清单（每屏必查）

- [ ] 所有触控目标 ≥ 48×48px
- [ ] 文字与背景对比度 ≥ 7:1（正文）/ 4.5:1（大字）
- [ ] 表单字段有可见 label，不依赖 placeholder
- [ ] 错误提示有图标 + 颜色 + 文字
- [ ] 焦点环可见，2px 主色实线
- [ ] 屏幕阅读器 aria-label 完整
- [ ] 支持 200% 浏览器缩放无横向滚动
- [ ] 不依赖颜色单独传递信息
